# 代码审查技能

当用户要求审查代码时，按照以下步骤进行。

## When to Use

✅ **USE this skill when:**

- "帮我看看这段代码"、"代码审查"
- "这个文件有没有 Bug"
- "优化一下这个函数"
- "检查下代码质量"

## When NOT to Use

❌ **DON'T use this skill when:**

- 用户只是要求读取文件内容、不涉及审查
- 用户要求编写全新功能（而非审查）

## Workflow

1. 首先使用 `read_file` 工具阅读目标文件
2. 从以下角度进行分析：
   - **代码质量**: 命名规范、函数长度、关注点分离
   - **潜在 Bug**: 空指针、边界条件、类型错误
   - **性能**: 不必要的循环、内存泄漏、缓存机会
   - **安全性**: 输入验证、SQL 注入、路径遍历
3. 给出具体的改进建议，必要时直接使用 `edit_file` 工具帮用户修改
4. 如果需要运行测试验证，使用 `bash_execute` 工具

## Output Format

回复格式建议：

```
🔍 代码审查：{文件名}

✅ 优点：
- ...

⚠️ 问题 & 建议：
1. [严重程度] 问题描述 → 建议修改
2. ...

📊 总评：{简短总结}
```
