# 晨间问候技能

当定时任务触发"晨间问候"或用户问你"今天怎么样"时，生成一段包含天气和待办的温馨问候。

## When to Use

✅ **USE this skill when:**

- 定时任务触发"晨间问候"
- 用户说"今天怎么样"、"早上好"、"早安"

## 核心原则

⚠️ **容错优先，必须发送**：即使部分信息获取失败，也**必须生成并发送问候消息**，不要反复重试失败的服务。

## Workflow

### 步骤 1：获取日期

```bash
date +"%Y年%m月%d日 %A"
```

### 步骤 2：获取天气（限时 10 秒）

优先使用 open-meteo API（国内服务器稳定），跳过 wttr.in（境内服务器经常超时）：

```bash
# 直接获取张家港天气（经纬度已知）
curl -s --max-time 10 "https://api.open-meteo.com/v1/forecast?latitude=31.865&longitude=120.53889&current=temperature_2m,relative_humidity_2m,weather_code,wind_speed_10m&daily=temperature_2m_max,temperature_2m_min,precipitation_sum&timezone=Asia%2FShanghai&forecast_days=1"
```

天气代码速查：0=晴☀️ | 1-3=多云⛅ | 45,48=雾🌫️ | 51-57=小雨🌦️ | 61-67=雨🌧️ | 71-77=雪❄️ | 80-82=阵雨🌦️ | 95-99=雷雨⛈️

**如果天气获取失败**：跳过天气部分，继续生成问候。

### 步骤 3：获取待办（可选，限时 5 秒）

使用 `search_memory` 工具搜索用户的备忘和待办。

**如果 TickTick 或其他待办服务不可用**：
- ❌ **不要反复重试**（最多尝试 1 次）
- ❌ **不要花时间查找脚本文件**
- ✅ 直接跳过待办部分，继续生成问候

### 步骤 4：生成问候

将收集到的信息组成温馨的问候消息，格式参考：

```
🌅 早安！今天是{日期}

🌤️ 天气：{天气描述}，{温度}°C
   （如果天气获取失败则省略此行）

📋 今日提醒：
- {待办事项1}
- {待办事项2}
   （如果无待办则写"今天没有特别的待办，轻松一天！"）

💪 {一句鼓励的话}
```

## 注意事项

- 整个问候生成不应超过 **30 秒**（包括所有 API 调用）
- **绝不要因为某个服务不可用而放弃发送问候**
- 微信消息不宜过长，控制在 300 字以内
- 鼓励的话可以根据天气、星期、节日等灵活变化
